## 3.2 Project Development

The development of PrimeLiving: Enhancing Apartment Management System with Real-time Notification and Task Organizations translated the approved designs and models into a working, web-based application. The system implements a four-tier role-based architecture comprising the System Administrator, Apartment Owner, Apartment Manager, and Tenant, each served by a dedicated dashboard and scoped set of permissions.

### System Administrator Dashboard

The system administrator dashboard serves as the top-level management workspace for overseeing the entire platform. Upon sign-in, the administrator is presented with an overview of key platform-wide indicators such as total registered apartment owners, active managers, and the number of pending inquiries. From this view, the administrator can manage apartment owner accounts by creating new owners through an email invitation flow, updating owner details, or deactivating accounts when necessary. The administrator also manages the inquiry lifecycle by reviewing submissions from the public-facing landing page, marking them as responded, approving them to create new owner accounts, or rejecting them. A search and filter interface allows the administrator to locate owner records quickly. Account settings enable the administrator to manage their own credentials.

### Apartment Owner Dashboard

The apartment owner dashboard serves as the primary property management workspace and was designed to consolidate the most frequently performed operational tasks into a single, coherent interface. Upon sign-in, the apartment owner is presented with an overview of key indicators such as total active tenants, pending maintenance requests, recent payment activity, and revenue summaries. From this view, the apartment owner can drill down to manage individual apartment properties, assign or remove apartment managers, add or deactivate tenants, and configure payment due dates per unit. The apartment owner oversees the entire payment lifecycle, including reviewing tenant-submitted payment receipts, approving or rejecting payment verifications, generating monthly billing records, and exporting payment data to Excel for record-keeping. Maintenance requests submitted by tenants are visible to the owner, who can track their progress through a structured status workflow from pending to in progress, resolved, and closed. The document management module allows the owner to upload and organize contracts, lease agreements, and house rules that become accessible to tenants and managers. The announcement feature enables the owner to broadcast notices to tenants and managers, with optional SMS delivery to targeted units. Revenue tracking and analytics provide financial visibility across all managed properties.

### Apartment Manager Dashboard

The apartment manager dashboard serves as the day-to-day operational workspace and was designed to streamline routine property management tasks within the scope of the manager's assigned apartments. Upon sign-in, the apartment manager is presented with an overview of assigned apartments, tenant counts, occupancy status, and a count of pending maintenance requests displayed as a notification badge. The manager can view assigned units and tenant assignments, review and update maintenance request statuses with notes, and verify tenant payment submissions by reviewing uploaded receipt images or QR code proofs. Search, sorting, and filter controls were implemented to locate records quickly. A notification inbox displays in-app messages with unread count badges, allowing the manager to stay current with system events. The manager operates within a scoped permission model and cannot delete data or access apartments outside their assignment.

### Tenant Portal

The tenant portal was implemented as a mobile-responsive, self-service interface to reduce in-person inquiries and improve transparency. After secure authentication, a tenant can view their current apartment and unit information, payment due dates, and recent transactions. The portal provides access to view billing schedules and submit payment proofs by uploading receipt images or QR code screenshots, with each submission tracked through a verification workflow where the manager or owner approves or rejects the proof. Tenants can submit maintenance requests with a title, description, priority level, and optional photo attachments to help managers identify issues appropriately. A documents section allows tenants to download or view lease agreements, contracts, and house rules that the owner has made available. A unified notifications center displays announcements and SMS alerts sent by the system, with unread count indicators to minimize missed reminders and enable tenants to track their communication history with the manager. Account settings allow tenants to update their password and personal information.

### Payment Tracking

Payment tracking in the system distinguishes between payment states such as pending, verified, and overdue. When payments are submitted, tenants upload proof of payment through receipt images or QR code screenshots. The apartment manager or owner then reviews and approves or rejects the verification. Monthly billing records are generated systematically, and the system tracks payment history per tenant with timestamps and status transitions. Payment data can be exported to Excel format for compliance and record-keeping. The billing module supports configurable payment due dates per apartment unit, and overdue balances are tracked to trigger notification reminders.

### Notifications

Notifications were implemented using an event-driven approach that triggers alerts based on business rules rather than manual interventions. Typical events include the creation of a new charge, upcoming due dates, overdue balance transitions, successful payment verification, and approaching contract expiry. The system integrates with Semaphore, a Philippine-based SMS gateway, through a secure API for SMS delivery, with delivery statuses recorded in SMS logs for monitoring and troubleshooting. Email serves as a secondary channel for account invitations, password resets, and account activation. In-app notifications provide a real-time inbox for each user role with unread count badges and mark-as-read functionality. Announcements created by owners or managers are delivered to targeted tenants with optional SMS dispatch. All notifications are logged to maintain an audit trail so both managers and tenants can verify what was sent and when.

### Maintenance Reporting

Maintenance reporting was developed as a structured ticketing workflow that channels tenant requests into an orderly queue. Tenants submit a request from the portal by providing a title, describing the issue, selecting a priority level, and optionally attaching photos. The system assigns a unique identifier and timestamps the request. The apartment manager or owner then acknowledges the ticket and updates its status through stages: pending, in progress, resolved, and closed, with the ability to add notes at each stage. Status changes are visible to the tenant, providing transparency into progress without requiring follow-ups by phone or chat. Completed tickets retain their full history, and the structured approach creates accountability, shortens resolution times, and yields historical data that can inform preventive maintenance planning.

### Document Management

The document management module allows apartment owners to upload, organize, and share formal documents such as lease agreements, contracts, house rules, and property-related files. Documents are stored securely through Supabase Storage and can be scoped to specific apartments or tenants. Tenants can view and download documents that have been made available to them, ensuring they always have access to current agreements and policies. The system maintains upload history and supports common file formats including PDF and images.

### Technology Implementation

PrimeLiving was developed with a modern, maintainable stack that balances performance with developer productivity. The front end uses React 18 with TypeScript to deliver a responsive single-page application with client-side routing through React Router, form validation powered by React Hook Form and Zod, and reusable UI components built with Radix UI primitives and styled with Tailwind CSS. Vite serves as the build tool for fast development iteration and optimized production bundles. Interactive dashboard charts and analytics are rendered using Recharts, while PDF generation for receipts and reports is handled client-side with jsPDF, and data exports use the XLSX library for Excel file creation. The back end was implemented in Express.js 5 with TypeScript, providing a structured REST API architecture with dedicated controllers, routes, and middleware for authentication, authorization, validation, and error handling. Supabase serves as both the authentication provider and the PostgreSQL database, with Row-Level Security policies enforcing data access at the database level. The database schema comprises thirteen interconnected tables covering apartment owners, managers, apartments, tenants, payments, maintenance requests, documents, announcements, revenues, inquiries, notifications, SMS logs, and apartment audit logs, with foreign keys and indices defined according to the entity relationship diagram to maintain referential integrity and query performance. For messaging, the application integrates with Semaphore, a Philippine-based SMS gateway, through secure API keys stored in environment variables. Email validation leverages the AbstractAPI service as an optional verification layer. Security practices including password hashing via Supabase Auth, role-based access control with four permission tiers, Helmet security headers, CORS configuration, input sanitization through Zod validation, and compression middleware were applied throughout to protect user data and maintain system integrity. Development utilized GitHub for version control and the project followed a component-driven development approach with the frontend and backend maintained as separate packages within a unified workspace.
